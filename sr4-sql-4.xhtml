Yay, another release!

Fifth Edition is here by now, and I promise, there will be a database for that one as well. (I wrote an initiative tracker for Fifth Edition, by the way. Expect another blog post about that one soon-ish.) I really like some of the changes, like what they did to Initiative, but I'm not so fond of other things - they could've done without the limits, in my opinion. I haven't played it yet - we're currently scheduling a play test, though. For now, I've been working on the Fourth Edition database a little, and I've got just a couple more features for you:

  • BP cost and tables for bound spirits
  • BP cost and tables for bound foci
  • Tables for spirits (skills, powers, spirit types and the like)
  • Support for Aptitude and Exceptional Attribute qualities
  • and a couple of bug fixes in BP cost computation
  • as well as fixed boundary checks for skills and skill groups
So now the magic users have got their toys as well as the technomancers - sorry for the delay, but good things come to those who wait, right? ;) I still need to do a lot of data entry, and character improvement still isn't implemented, but the basic rules for BP character generation should all be covered by now.

So how do I use this?

I'm working on a website for this database right now; check out its early stages at http://sr4.ef.gy. However, that website is still under construction, so please don't expect it to be all that usable just yet - when it is, I'll post about it on the blog. You're welcome to play around with it and make suggestions for the interface, but don't expect anything complete or working just yet. Feel free to grab the database and write your own front end, web based or not; if you do that and tell me about your project I'll be happy to link to it.

The public git repository for sr4.sql is here: http://git.becquerel.org/?p=machinelady/sr4.sql.git, if you'd like to browse the source code or grab a tarball; or you could clone the repository using 

$ git clone git://git.becquerel.org/machinelady/sr4.sql.git

, which will create a directory named 'sr4.sql'. Run sr4.sh to concatenate the SQL files and feed them to SQLite; if you'd like to add user-generated content, I put all the data about spells, gear and the like into sr4data.sql, so you could edit that file or create your own.
