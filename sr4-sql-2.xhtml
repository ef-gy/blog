So I'm still working on that thing...

With the PDF release of 5th edition just around the corner, you'd think it's obsolete, but I'm looking forward to checking out Fifth Edition and translating it to SQL, too... well, the latter, not so much. It's pretty clear the rules are designed to be interpreted by humans, not computers, so the SQL is getting more complex and bloated than I'd like. I heard they streamlined the rules quite a bit in Fifth Edition though, so this bodes well for sr5.sql.

Anyway I've been working on the Fourth Edition version, as time permitted, and I've got a couple of new features for you:

  • BP cost for skill specialisations
  • BP costs for spells and complex forms
  • BP costs for registered sprites at character generation
  • Skill groups
  • Essence cost computation
  • Reduction of maximum Magic and Resonance attribute by Essence loss
  • native languages for characters
Don't worry, magic users - your foci and bound spirits are still on my list. You're gonna get your toys, it's just that I'm quite fond of technomancers, so they got theirs first in this release. It's your turn in the next one though. By the way, for the time being this database only works correctly for starting characters - character advancement is on my list, so you'll eventually be able to maintain a current copy of your shadowrunner through the database, but right now all you can do is create a new one.

So how do I use this?

For now, there's really just the database, no GUI or CLI front end yet. A web-based front end is planned; I'm probably going to start working on it after this release. Feel free to grab the database and write your own front end, web based or not; if you do that and tell me about your project I'll be happy to link to it.

The public git repository for sr4.sql is here: http://git.becquerel.org/?p=machinelady/sr4.sql.git, if you'd like to browse the source code or grab a tarball; or you could clone the repository using 

$ git clone git://git.becquerel.org/machinelady/sr4.sql.git
, which will create a directory named 'sr4.sql'. Run sr4.sh to concatenate the SQL files and feed them to SQLite; if you'd like to add user-generated content, I put all the data about spells, gear and the like into sr4data.sql, so you could edit that file or create your own.
