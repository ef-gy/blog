Introduction

Topologic, your friendly Mac App Store App to visualise higher-dimensional geometric primitives, is currently getting a series of cleanups. Part of these cleanups is a CLI programme that can't display these primitives, but it can read and generate the SVGs produced by the App. In other words, you get to use your command line to type topologic --model cube --depth 4 --render-depth 4 > 4-cube.svg, and that results in this pretty thing:

Also works with simplices, spheres, moebius strips and klein bagels.

Update (2013-10-10): Release 6 is out now, which includes more cleanups, an open source GLUT client, majour improvements to the mouse handling and sierpinski gaskets and carpets.

Build Requirements

To compile the programme, you'll need a current version of clang++ (with partial C++0x support), libxml2 with development headers and this site's maths templates, libefgy.

It may be possible to compile this programme with g++, but I've never actually tried that.

Compiling

First you're going to need the source code, either grab this source tarball for release 6, or grab the sources with git at http://git.becquerel.org/jyujin/topologic.git. As usual you can browse the sources via gitweb at http://git.becquerel.org/jyujin/topologic.git. Then you need to untar whatever you got and do something like this in a shell:

$ cd libefgy-1
$ make
$ sudo make PREFIX=/usr install

Substitute sudo as appropriate. This should install the binary in /usr/bin/topologic and the related header files in /usr/include/topologic. The latter is somewhat important, as the programme is highly template-y.

Oh yeah, do note the MIT/X-style licence in the COPYING file and browse through the README while you're at it.

Update (2013-10-10): use make topologic-glut to build the GLUT frontend. This is a separate frontend that is not build by default because it requires OpenGL and GLUT libraries to be installed. Usage is exactly the same as the regular CLI frontend, but the output is rendered in a new OpenGL window instead of as an SVG to stdout.

Manual

The programme supports the default --help switch to show a summary of the available options. Better yet, it comes with a fairly elaborate man page that tries to explain all about how to use the programme. You really ought to read that.
